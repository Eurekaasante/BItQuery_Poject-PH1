## 1. Problem Statement
A retail dataset to test BitQuerry profieciency is to be analysed to generate insights. 
## 2. Intention of the Exercise
The primary goal is to evaluate the students' technical agility and analytical depth within BItQuerry coding platform. Specifically, the exercise intends to:

* Test the ability to translate business requirements into technical data queries.
* Assess proficiency in navigating the Bitquery GraphQL IDE and its unified schema.
* Validate the student's understanding of relational data logic (SQL functions).
* Ensure students can document technical findings into a professional, stakeholder-ready PDF report.

## 3. Methodology
The assessment follows a four-stage execution path:

* Logical Mapping (SQL Foundations): Students first complete a questionnaire that identifies the underlying logic of the investigation (e.g., identifying which aggregate functions to use for volume). This ensures they understand the math before the code.
* Technical Execution (GraphQL Querying): Using the Bitquery IDE, students must construct and run live queries. They are required to navigate various apects of the platform to extract real-time data for the BrightTV contract.
* Verification (Proof of Work): For every result obtained, students must take a screenshot showing the query and the JSON response side-by-side.
* Synthesis (Consolidated Reporting): The methodology concludes with the compilation of all SQL answers, GraphQL queries, and result screenshots into a single, indexed PDF file. This tests the student’s ability to present complex data in a structured, readable format.

## 4. Intended Outcomes
By completing this exercise, students will demonstrate:

*Data ingestion
*Creation of tables, schemas and projects.
* Schema Fluency: The ability to find and filter, manipulate and analyse from the codes generated.
* Operational Efficiency: Writing optimized queries that respect requesting only the data necessary for the retail case.
* Data Interpretation: Moving beyond "pulling data" to explaining what the data says about etaildata set.
* Professional Documentation: A polished final PDF that serves as an evidentiary record of their technical skill level.

## 5. Conclusion
This exercise serves as a comprehensive litmus test for modern data roles. By using the retail sales data, students prove they can handle the "noise" of big datasets and extract "signal" using Bitquery’s powerful engine. Successful completion indicates that a student is not just a coder, but a capable analyst ready to support decentralized organizations in making data-driven decisions.
